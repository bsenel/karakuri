# Reflexion vs Chain-of-Thought benchmark

_Generated by `cmd/krk-bench` on 2026-05-22. Seed 42, 200 trials/scenario, max 8 iterations._

## Methodology

Each scenario has a `difficulty` parameter — the probability that a single 
agent call produces a plan whose verify step passes. Chain-of-thought uses 
that probability directly. Reflexion applies a 40% multiplicative boost to 
(1-difficulty) — i.e. the critique-and-revise pass closes 40% of the gap to 
certainty per iteration. This is a synthetic benchmark; real-LLM numbers 
will vary with provider and prompt design.

Call cost: Reflexion is charged 3 calls per iteration (draft + critique + 
revise); chain-of-thought is charged 1. Both strategies use the same `max-iter`.

## Interpretation

Reflexion converts call budget into a higher success rate on hard 
scenarios (difficulty ≤ 0.5) but spends more calls per iteration. The 
tradeoff is worthwhile when each call is cheap relative to the cost of 
an unmet objective (manual escalation, repeated retries). Set 
`agent.reasoning_strategy: reflexion` on agents that handle high-stakes 
tasks; leave the default `chain_of_thought` on routine ones.
